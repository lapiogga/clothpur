## 3. 테스트 중 발견된 이슈

### ISSUE-01: 포인트 잔여 음수 발생 (수정 완료)

| 항목 | 내용 |
|------|------|
| 발견 경위 | 1차 실행 시 T5-03 실패 |
| 원인 | seed 스크립트에서 같은 사용자에게 여러 pending 주문을 생성할 때 포인트 잔액 체크 없이 reserved_points 누적 |
| 영향 | 1개 레코드: `total=300,000`, `used=250,000`, `reserved=405,000` → 잔여 `-355,000` |
| 조치 | DB 직접 수정 (`reserved_points = MAX(0, total - used)`) + seed 스크립트에 `available >= totalAmount` 조건 추가 |
| 상태 | 수정 완료, 재테스트 통과 |
